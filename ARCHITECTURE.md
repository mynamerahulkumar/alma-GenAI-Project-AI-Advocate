## ğŸ“‹ Overview

This document outlines the architecture design for the Retrieval-Augmented Generation (RAG) Chatbot system. The system combines document retrieval with Large Language Models (LLMs) to provide accurate, context-aware responses based on ingested knowledge.

## ğŸ¯ System Goals

- **Transparency**: Ability to cite sources for generated answers

### Document Processing
| Parameter | Default | Description |
|-----------|---------|-------------|
| chunk_size | 500 | Maximum tokens per chunk |
| chunk_overlap | 50 | Overlapping tokens between chunks |
| separators | ["\n\n", "\n", ".", " "] | Text splitting priorities |

### Embeddings
| Parameter | Default | Description |
|-----------|---------|-------------|
| model_name | all-MiniLM-L6-v2 | HuggingFace model identifier |
| dimension | 384 | Embedding vector dimension |
| batch_size | 32 | Batch size for encoding |

### Vector Store
| Parameter | Default | Description |
|-----------|---------|-------------|
| index_type | FAISS | Vector store backend |
| metric | cosine | Similarity metric |
| top_k | 3 | Default retrieval count |

### RAG Chain (Future)
| Parameter | Default | Description |
|-----------|---------|-------------|
| llm_provider | Groq | LLM service provider |
| llm_model | llama-3.1-70b | Model for generation |
| temperature | 0.0 | Response randomness |
| max_tokens | 1024 | Maximum response length |
